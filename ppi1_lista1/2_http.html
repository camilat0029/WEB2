Protocolo HTTP

O HTTP(Protocolo de Transferência de Hipertexto) é usado para carregar páginas web usando os links de hipertexto. Este protocolo funciona em um modelo de comunicação que utiliza uma arquitetura Cliente-Servidor. Assim, quando o cliente faz uma solicitação ao servidor, este protocolo é utilizado nesta comunicação, logo o HTTP é a base da internet.

Requisição

O cliente ao fazer uma requisição ao servidor, envia-se uma mensagem HTTP, que podem ter ações como por exemplo GET ou POST. E o retorno do servidor após processá-la é uma response HTTP.

No caso dos servidores WEB, o retorno é um código HTML como uma página.

Exemplos de Requisição

  • GET: busca dados, solicita o perfil de usuários;
  • POST: envia dados para criar um novo usuário;
  • PUT: atualiza as informações/dados de um usuário;
  • DELETE: remove dados de um usuário

Request e response na Web

O request é a solicitação e o response é a resposta

Desta forma, o navegador envia um request ao servidor, e o servidor processa esta solicitação e após isso o servidor envia então uma response.

Assim, o cliente, navegador envia um método HTTP, como o GET ou POST, com a url, cabeçalhos, entre outros. E o servidor vai então processar isso e envia uma response com códigos de status, cabeçalhos e a resposta do que foi solicitado. Eles são de extrema importância na comunicação da Web.
